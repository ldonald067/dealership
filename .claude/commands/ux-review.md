Review a page's UX/UI against the project's design goals. If a specific page is mentioned, review that page. Otherwise, ask which page to review.

Design goals to evaluate against:
- Warm, premium, human feel — NOT generic SaaS/tech-bro aesthetic
- Think "premium car brand website" not "enterprise dashboard"
- Warm color palette (no cold blue/gray defaults)
- Rounded corners, subtle gradients, warm shadows
- Friendly, modern typography
- Aspirational car imagery, not clinical
- Reassuring status indicators, not corporate
- Subtle animations and transitions for delight
- Customer portal = luxury experience, dealer portal = polished but functional

Steps:
1. Ensure dev server is running
2. Use preview MCP tools to screenshot the page
3. Critique the page against each design goal above
4. Rate the page 1-10 on: warmth, premium feel, clarity, trust, delight
5. Give 3-5 specific, actionable improvements with code suggestions
6. Highlight anything that feels "generic SaaS" and suggest alternatives
